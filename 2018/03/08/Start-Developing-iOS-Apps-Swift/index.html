Start Developing iOS Apps (Swift) 精简版

发表于 2018-03-08 | 分类于 Swift |

Getting Started

Jump Right In

开始开发iOS应用程序(Swift)是学习创建在iPhone和iPad上运行的应用程序的完美起点. 下面的每课包含一个教程和完成它所需的概念信息. 这些课程相互构建,引导逐步创建简单真实世界的iOS应用程序.在完成课程并构建应用程序的过程中,将了解iOS应用程序开发中的概念,深入了解Swift编程语言,并熟悉Apple集成开发环境Xcode的许多有价值的功能(IDE).

Prerequisites/先决条件

在这些课程中,假定熟悉Swift programming language. 不需要成为 Swift master来完成课程,但如果能够舒适地阅读和理解Swift代码,你将会得到更多的收获.如果对Swift不太熟悉,在Swift Playgrounds应用程序中完成Learn to Code练习. 或者,可以通过The Swift Programming Language (Swift 4.1).的A Swift Tour. 两者都提供了Swift编程语言的坚实基础.

About the Lessons/关于课程

在课程中,将构建一个名为FoodTracker的简单膳食跟踪应用程序.这个应用程序显示了一份膳食列表,包括膳食名称,评分和照片.用户可以添加,删除或编辑膳食. 要添加新餐或编辑现有餐,用户可以导航到不同的屏幕,在这里他们可以指定特定餐点的名称,评分和照片.

这些课程每个都附带一个Xcode项目文件,该文件显示代码和界面应该如何看待课程结束的示例. 在完成课程后,可以下载项目并检查工作.如果需要参考整个课程中学到的概念,使用glossary来刷新记忆. Glossary术语贯穿整个课程.

Get the Tools/获取工具

要使用这些课程中描述的最新技术开发iOS应用程序,需要运行最新版本Xcode的Mac计算机(macOS 10.11.5或更高版本). Xcode包含设计,开发和调试应用程序所需的所有功能. Xcode还包含iOS SDK,它扩展了Xcode以包含专门用于iOS开发的工具,编译器和框架.

Building the UI

Build a Basic UI

本课可以熟悉Xcode,用于编写应用程序的工具.熟悉Xcode中项目的结构,并学习如何在基本项目组件之间进行导航和使用. 在本课中,将开始为FoodTracker应用程序制作一个简单的user interface (UI)并在模拟器中查看它. 完成后,应用将会有一个餐名的标签,一个用于更改餐名的文本字段以及一个用于重置名称的按钮.

Learning Objectives/学习目标

在本课结束时,将掌握:

  1. 在Xcode中创建一个project
  2. Identify the purpose of key files that are created with an Xcode project template
  3. 在项目中打开并切换文件
  4. 在iOS模拟器中运行应用程序
  5. 在storyboard中添加,移动和调整UI elements 的大小
  6. 使用属性检查器(The Attributes inspector)编辑storyboard中UI elements的属性
  7. 使用大纲视图(The Outline view)查看和重新排列UI elements
  8. 使用助手编辑器(The Assistant editor’s)的预览模式预览storyboard UI
  9. 使用自动布局(Auto Layout)自动适应用户设备的尺寸大小

Create a New Project

Xcode包括几个内置的应用程序模板,用于开发常见类型的iOS应用程序,例如游戏,基于标签导航的应用程序和基于表格视图的应用程序. 大多数这些模板都具有预配置的界面和源代码文件. 先从最基本的模板开始:Single View Application.

To create a new project

  1. Open Xcode from the /Applications directory.
  2. 在欢迎窗口中,点击“创建一个新的Xcode项目”(或者选择File> New> Project).Xcode打开一个新窗口并显示一个对话框,可以在其中选择一个模板.
  3. 在对话框顶部选择iOS.
  4. 在应用程序部分中,选择单一视图应用程序,然后单击下一步.
  5. 在出现的对话框中,使用以下值命名应用程序并为项目选择其他选项:
    • Product Name: FoodTracker.Xcode使用输入的产品名称命名项目和应用程序.
    • Team: 如果未自动填充,将团队设置为无.
    • Organization Name:组织的名称或自己的名字.可以留下这个空白.
    • Organization Identifier: 组织标识符,如果有的话. 如果不这样做,使用 com.example.
    • Bundle Identifier: 该值根据产品名称和组织标识符自动生成.
    • Language: Swift
    • Devices: Universal.用应用程序可以在iPhone和iPad上运行.
    • Use Core Data: Unselected.
    • Include Unit Tests: Selected.
    • Include UI Tests: Unselected.

  1. 点击下一步.
  2. 在出现的对话框中,选择一个位置来保存项目并点击创建.

工作区窗口可能会显示一个错误图标,其中显示“签署FoodTracker需要一个开发团队”.此警告意味着尚未为iOS开发设置Xcode,不用担心,这个课程不需要开发团队在模拟器中运行应用程序.

Get Familiar with Xcode

Xcode包含创建应用程序所需的一切. 它组织所有创建应用程序的文件和资源. 它为代码和用户界面提供编辑器. 此外,Xcode允许构建,运行和调试应用程序 - 为iOS设备提供模拟器和强大的集成调试器.

熟悉一下Xcode工作区的主要部分:

  • Navigator area. 提供对项目各个部分的快速访问.
  • Editor area. 允许编辑源代码,用户界面和其他资源.
  • Utility area. 提供有关选定项目的信息和对现成资源的访问. 实用区域分为两部分. 顶部是检查器窗格,可以在其中查看和编辑有关在导航器或编辑区域中选择的项目的信息. 底部是库窗格,可以在其中访问用户界面元素,代码片段和其他资源.
  • Toolbar. 用于构建和运行应用程序,查看正在运行的任务的进度以及配置工作环境.

Run iOS Simulator

由于是基于Xcode模板构建项目的,因此基本应用程序环境会自动设置. 即使没有编写任何代码,也可以构建并运行单一视图应用程序模板,而无需任何其他配置.

要构建和运行应用程序,使用Xcode中包含的iOS模拟器应用程序.通过模拟器可以了解应用程序在设备上运行时的外观和行为.

模拟器可以模拟许多不同类型的硬件 - 所有的屏幕尺寸和分辨率适用于iPad和iPhone - 这样可以在每个正在开发的设备上模拟您的应用程序. 在本课中,使用iPhone 7选项.

To run your app in the simulator

  1. 在Xcode工具栏的Scheme弹出菜单中,选择iPhone 7.

“Scheme”弹出式菜单可选择要运行应用程序的模拟器或设备. 确保选择了iPhone 7模拟器,而不是iOS设备.

  1. 点击位于Xcode工具栏左上角的运行按钮.

    或者,选择Product> Run(或按Command-R).如果第一次运行应用程序,Xcode会询问是否希望在Mac上启用开发人员模式. 开发者模式允许Xcode访问某些调试功能,而无需您每次输入密码. 决定是否要启用开发人员模式并按照提示进行操作.

    如果选择不启用开发人员模式,则稍后可能会要求输入密码. 这些课程假设开发者模式已启用.

  2. 在构建过程完成时观看的Xcode工具栏.

    Xcode在活动查看器中显示有关构建过程的消息,该消息位于工具栏的中间.
    Xcode完成构建项目后,模拟器会自动启动. 第一次启动可能需要一些时间.
    模拟器以指定的iPhone模式打开,然后启动应用程序. 最初,模拟器显示应用程序的启动屏幕,然后转换到应用程序的主界面. 在未修改的Single View Application模板中,启动屏幕和主界面完全相同.

目前,单一视图应用程序模板并没有太多 - 它只是显示一个白色屏幕. 其他模板具有更复杂的行为. 在扩展它来制作自己的应用程序之前,了解模板的用途很重要. 在没有修改的情况下在模拟器中运行应用程序是开始理解的好方法.通过选择模拟器>退出模拟器(或按Command-Q)退出模拟器.

Review the Source Code

Single View Application模板附带一些设置应用程序环境的源代码文件. 首先,看看AppDelegate.swift文件.

查看AppDelegate.swift源文件

  1. 确保项目导航器已在导航区域中打开.

  2. 如有必要,点击项目导航器中FoodTracker文件夹旁边的显示三角形.

  3. 选择AppDelegate.swift.

The App Delegate Source File

AppDelegate.swift源文件有两个主要功能:

  • 它定义AppDelegate类. 应用程序委托创建应用程序内容的绘制窗口,并提供一个位置来响应应用程序中的状态转换.

  • 它为应用程序创建入口点,并为应用程序提供输入事件的运行循环. 该工作由UIApplicationMain属性(@UIApplicationMain)完成,该属性出现在文件顶部.

    使用UIApplicationMain属性相当于调用UIApplicationMain函数并将AppDelegate类的名称作为委托类的名称传递. 作为响应,系统创建一个应用程序对象. 应用程序对象负责管理应用程序的生命周期. 系统还会创建AppDelegate类的实例,并将其分配给应用程序对象. 最后,系统启动应用程序.

每当创建一个新项目时,AppDelegate类都会自动创建. 除非正在做非常不寻常的事情,否则使用Xcode提供的此类来初始化应用程序并对应用程序级别的事件做出响应. AppDelegate类采用UIApplicationDelegate协议. 该协议定义了许多用于设置应用程序,响应应用程序状态更改以及处理其他应用程序级事件的方法.

AppDelegate类包含一个属性:窗口.

该属性存储对应用程序窗口的引用. 此窗口代表您的应用视图层次结构的根. 这是您所有应用内容的绘制位置. 注意,window属性是可选的,这意味着在某个点它可能没有值(nil).

这些方法让应用程序对象与应用程序委托进行通信. 在应用程序状态转换过程中(例如,应用程序启动,转换到后台和应用程序终止),应用程序对象会调用相应的委托方法,为应用程序提供一个响应的机会.

每个委托方法都有一个默认行为. 如果将模板实现留空或从AppDelegate类中将其删除,则无论何时调用该方法,都会获得默认行为. 或者,可以将自己的代码添加到stub方法,以定义在调用方法时执行的自定义行为.

该模板还为每个stub方法提供注释. 这些注释描述了这些方法如何被应用使用. 可以使用stub方法和注释作为设计许多常见应用程序级行为的引导.
