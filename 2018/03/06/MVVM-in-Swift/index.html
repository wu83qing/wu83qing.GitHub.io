MVVM in Swift

发表于 2018-03-06 | 分类于 Swift |

引言

MVVM是Model-View-ViewModel的缩写,听起来很复杂,但它本质是MVC模式的改进版.MVVM 就是将其中View 的状态和行为抽象化,让我们将视图 UI 和业务逻辑分开.当然这些事 ViewModel 已经帮我们做了,它可以取出 Model 的数据同时帮忙处理 View 中由于需要展示内容而涉及的业务逻辑.

一般来说,MVVM通常与Functional Reactive Programming(FRP)结合使用,并且有很多FRP库,例如RxSwift和ReactiveCocoa.但是,如果有人不熟悉FRP,那么怎么在项目中使用MVVM呢.在本文中,将演示在没有FRP库的情况下使用MVVM.

Model View Controller

先开始使用标准的MVC方式.例子是在UITableView中显示一个排序后的integers Array(整数数组).另外,通过点击右上角的添加按钮,可以按照正确的顺序插入一个新的整数,并通过操作cell来删除一个integer.

在这里,先创建一个名为DemoViewController的UITableViewController子类,并在其中实现必要的UITableViewDataSource方法.另外,将插入逻辑放在addNewInteger方法中,将删除逻辑放在tableView(_tableView:,commit editingStyle:,forRowAt indexPath:)方法中.

Model View View Model

转换为MVVM的第一步是将其中View 的状态和行为抽象化.在我们例子中为一个排序的Integers table View,因此我们对table View的状态和行为进行处理,创建一个名为State的 struct,它存储table View的状态和行为.
首先在插入或删除一个integer后应该更新表视图,所以先编写一个名为EditingStyle的 emun,它包含insert,delete和none情况.
此外在State中创建一个editingStyle属性,并使用editingStyle属性上的Swift属性观察器功能(Swift’s property observers)更新sortedIntegers属性.

现在,根据这个思路,我们很容易创建View Model类,它包含插入和删除逻辑.此外,我们的View Controller只能通过我们的View Model访问状态信息.如果状态信息发生变化,我们的View Controller也可以通过View Controller的回调闭包来更新其表视图.

最后,修改视图控制器内部的代码以将所有内容都绑定起来. 创建一个viewModel属性并在viewDidLoad方法中实例化它.

总结

在项目中采用MVVM有几个优点.
首先,它使代码库更易于测试.Apple 的View Controller总是不方便进行集成测试,采用MVVM模式可以将大部分业务逻辑迁移到视图模型中为业务逻辑编写测试提供了便利性和可能性.
其次,遵循这种模式可以使代码库更加一致并带来更多的可读性.
此外,可以进一步降低与FRP库中的UI binding(UI 绑定)和Asynchronous Chaining(异步链接)的复杂性.
