基本介绍

SfChart基于MPAndroid,主要用于绘制各种图表,是先有了Android版本后才开发的iOS版本,两版本的API设计方式都是相同的,且以Android为准.因此SfCharts中组件代码架构可以根据MPAndroid架构来映射.主要分为LineData、Axis、MakerView、ViewPort.

先看一个基本图表所包含的元素.

  • 黄色区域:YAxis 负责Y轴的样式,如:刻度间距,显示刻度数量,背景横线,显示横向LimitLine
  • 绿色区域:XAxis 负责x轴的样式,如:刻度间距,显示刻度数量,背景横线,显示直向LimitLine
  • 紫色区域:Legend 负责图例的样式,如 图例形状 行距 间距 自定义颜色
  • 黑色箭头:LineData 图表数据与一些样式设定, 如 数据集合 线条样式 数据点样式

  • 橘色圆形:MakerView 点击数据点会弹出个Maker

    其中最重要的就是LineData,如果图表没有LineData数据,就只能单纯显示(No Chart data available)的文字,所以选择好图表类型并吧Chart放入Layout 第一步是将数据放入Chart中.

How

以折线图 LineChart 来做范例进行模型拆分.

Data 数据源

Chart用LineData座位封装所以数据的实体,并有DataSet Entry XVals等不同的元素,以下图LineChart来说明基本概念;

LineData 中的 List 有两个 DataSet,每个 DataSet 都是一个类别,并由 List 所组成,简单来看,数据的组成如下:

DataSet/Entry Entry0 Entry1 Entry2 Entry3 Entry4
DataSetA 0 2 4 6 8
DataSetB 0 4 8 12 16
  • 图右则是实际代码使用的Method 代表设定数据时的结构 Entry是最小模型 负责记录数据列的值与顺序.
  • 图左下发绿色区域的XVals则表示X轴的坐标Label 是一个List

有了基本概念,可以先构建一个LineData

LineData的Base Class 是ChartData 用于封装所有Data与 Styling Data,而不同图表的类型,有不同的SubClass.比如柱状图是使用的 BarData 、BarDataSet、BarEntry等都是继承ChartData,虽不同组件,但实际上用法大同小异,可以在ChartDataConfig.js查看组件的结构.

Data Color

设置此数据类型的基础颜色




Data Point

设定每个 Entry 出现点的样式

lineData.dataSets.config.drawCircles 预设为 True,可以直接设定 Data Point 的样式

lineData.dataSets.config.drawCircleHole

False = 实心的单一颜色

True = 如范例图并可以设定 CircleColorHole的颜色

Line

设定画出线条的样式

Highlight

Highlight 是当点下 DataPoint 的时候,会在背景 Grid 线上出现一条用来对齐数值的指引线,预设是水平跟垂直都会显示

设定线条宽度与颜色

如果要设置只有垂直或水平单一条 Highlight 的话,可以先lineData.dataSets.config.common.highlightEnabled 设为 False,再通过drawVerticalHighlightIndicator 或

drawHorizontalHighlightIndicator 来打开垂直或者水平 Hightlight 线


Fill

在折线图的一个data类别中,把data线下的面积填满

lineData.DataSets.config.lineRadar.drawFilled 预设是 False,要打开才会有 Fill 作用,而最后加入的 dataset ,他的 z-index 为最大,会显示在最上层

Position

  1. TOP
  2. TOP_INSIDE
  3. BOTTOM（预设）
  4. BOTTOM_INSIDE
  5. BOTH_SIDED

Label

设定 Label 显示方式,以下介绍主要 Label 的设定：

文字的设定分为 Color、Size

Line

设定 mXAxis 的线条样式,有分两种：

GridLine 是负责画出背景格线中的所有垂直线,可以设定 Color、Width、DashedLine

AxisLine 设定 mXAxis 最靠近数值 Label 的水平线

如果不想显示线条的话,也可以选择关闭

YAxis Y轴

YAxis 因为由 Right 跟 Left 两条线组成,所以会影响到 Y 轴 Label 数值的设定,大部分都需要两边同时设定,否则两边显示的数值会不一样,或是 Label 数值线没有对齐等状况

Max/MinValue

设定 mYAxis 显示 Label 数值的最大值与最小值

需要先把 yAxis.left(right).zeroLine 设为 False,才可以设定上下限

Order

设定 Lebel 数值要 ASC 或 DESC 排序

Space

Space 是设定data之最大值与最小值的向外增加空间

Top 为正值所设定多出来的向上空间

Bottom 为负值所设定多出来的向下空间

Position

决定 Label 要显示在 Chart 内还是外,有两种：

  1. OUTSIDE_CHART（预设）
  1. INSIDE_CHART

Count

yAxis.left(right).labelCount：设置显示几个 Label 数,

yAxis.left(right).labelCountForce 第二个选项 Force,有一些差异：

  1. Ture ：代表会精确的平均数值,均匀分配 Label

Interval = [(Max - Min) / (Count - 1)]

  1. False：以算出间距来慢慢加上数值,不一定均匀分配 Label

Interval = 透过 [(Max - Min) / Count] 算出 Range 后,再经过 log 跟小数的处理后得出

差异在于这个方法不一定会显示 chart 顶端的最大数值,没有设定的话,一般图表预设都是以这个方式显示

一般没有设定的话,Label 数量 Default = 6,最大可以设定 25 个

Line

设定 mYAxis 的线条样式,有分两种：

GridLine负责画出图表背景格线的所有水平线

AxisLine 为最靠近数值 Label 的垂直线

如果不想显示线条的话,也可以选择关闭
