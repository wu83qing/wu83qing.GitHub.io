UIKit框架

发表于 2017-06-06 | 分类于 iOS

官方文档链接

UIKit 框架为您的iOS或tvOS应用程序提供了所需的基础架构. 它提供了用于实现界面的窗口和视图体系结构,用于向应用程序提供多点触控和其他类型输入的事件处理基础结构,以及管理用户,系统和应用程序之间的交互所需的主运行循环. 该框架提供的其他功能包括动画支持,文档支持,绘图和打印支持,关于当前设备的信息,文本管理和显示,搜索支持,可访问性支持,应用程序扩展支持和资源管理.

  • First Steps
    • About App Development with UIKit
      了解UIKit和Xcode为您的iOS和tvOS应用程序提供的基本支持.
  • App Structure
    • Core App
      管理您的应用的数据模型及其与系统的交互.
    • Resource Management
      管理您存储在主要可执行文件之外的 images, strings, storyboards, and nib files
    • App Extensions
      将应用程序的基本功能扩展到系统的其他部分.
  • User Interface 用户界面
    • Views and Controls
      在屏幕上显示您的内容,并定义与该内容相关的交互.
    • View Controllers
      使用视图控制器管理你的界面,并促进周围的应用程序的内容导航.
    • View Layout
      使用堆栈视图自动布置界面的视图. 当您需要精确放置视图时,请使用自动布局.
    • Animation and Haptics 触觉
      使用基于视图的动画和触觉向用户提供反馈.
    • Windows and Screens
      为您的视图层次结构和其他内容提供一个容器.
  • User Interactions 用户交互
    • Touches, Presses, and Gestures 触摸,按压和手势
      在手势识别器中封装您的应用程序的事件处理逻辑,以便您可以在整个应用程序中重复使用该代码.
    • Drag and Drop 拖放
      通过使用交互API和视图,拖放到您的应用程序.
    • Focus Interactions 焦点互动
      使用远程或游戏控制器导航UIKit应用程序的界面.
    • Peek and Pop
      使用3D触摸输入来显示您的内容的自定义预览和操作.
    • Keyboard and Menus 键盘和菜单
      处理键盘输入,并显示自定义操作菜单.
    • Accessibility 无障碍
      让残障用户更容易使用您的应用.
  • Graphics, Drawing, and Printing
    • Images and PDF
      创建和管理图像,包括使用位图和PDF格式的图像.
    • Drawing
      使用渲染器配置应用程序的绘图环境,并绘制路径,字符串和阴影.
    • Printing
      显示系统打印面板并管理打印过程.
  • Text
    • Text Display and Fonts
      使用UIKit视图来显示文本,管理字体和检查拼写.
    • Text Storage
      管理文本存储,并协调文本的布局.
    • Keyboards and Input
      配置系统键盘,或创建自己的键盘和自己处理输入.

Apple 编程核心思想

发表于 2017-05-06 | 分类于 iOS

面向对象与面向过程

  • OC语言是面向对象,C语言是面相过程.它们是解决问题的两种思想.
  • 两者的不同点是:面向过程管住的是解决问题需要那些步骤,面向对象关注的是解决问题需要那些对象.
  • 面向对象的核心是对象,面向对象有两个特别重要的概念:类和对象.
  • 编程语言的对象建立,是通过类来创建.就相当于盖房子用的设计图纸,类就是创建对象的描述.对象是类的具体存在,一般在解决问题中都会先设计类,再通过类创建相应的对象.
  • 类的创建有三个属性:类名、属性、行为(功能),一般名词就是类名,切首字母必须大写,不能使用下划线.

面向协议编程(Protocol Oriented Programming,以下简称 POP)是 Apple 在2015年WWDC上提出的Swift的一种编程范式.
相比与传统的面向对象编程(OOP),POP 显得更加灵活.
结合 Swift 的值语义特性和 Swift 标准库的实现,这一年来大家发现了很多 POP 的应用场景.

Protocol
Swift 标准库中有 50 多个复杂不一的协议,几乎所有的实际类型都是满足若干协议的.protocol 是 Swift 语言的底座,语言的其他部分正是在这个底座上组织和建立起来的.这和我们熟知的面向对象的构建方式很不一样.

但是不是能通过OOP对真实世界进行全部的模拟建模呢?答案是否定的,虽然我们努力使用继承与封装的思想来进行建模,但实际事物往往是一些列特质的组合,而不是单单的一脉继承,并逐渐扩展的方式开展的.
