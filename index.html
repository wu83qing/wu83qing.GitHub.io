About Swift

Swift是编写软件的绝妙方式,无论是用于手机,台式机,服务器还是其他运行代码的软件. 这是一种安全,快速和交互式的编程语言,融合了现代语言思维的最佳水平和更广泛的Apple工程文化的智慧以及开源社区的多元贡献. 编译器针对性能进行了优化,并且语言针对开发进行了优化,而无需牺牲任何一种.

Swift对新程序员很友好. 它是一种工业品质的编程语言,与脚本语言一样富有表现力和愉悦感. 在playground中编写Swift代码可让试验代码并立即查看结果,而无需构建和运行应用程序.

Swift通过采用现代编程模式来定义大量常见编程错误:

  • 变量总是在使用前初始化.
  • Array索引检查出界错误.
  • 检查Integers是否溢出.
  • 选项确保nil被明确处理.
  • 内存自动管理.
  • error处理允许从意外故障中控制恢复.

Swift代码被编译和优化,以充分利用现代硬件.语法和标准库的设计基于指导原则,写代码的显而易见的方式也应该表现得最好. 它的安全性和速度相结合使Swift成为从“Hello,world！”到整个操作系统的绝佳选择.

Swift将强大的类型推理和模式匹配与现代轻量级语法相结合,使复杂的想法以清晰简明的方式表达出来. 因此,代码不仅易于编写,而且易于阅读和维护.

Swift已经有多年的历史了,并且它不断发展,具有新的特性和功能. Swift目标是雄心勃勃的.

Version Compatibility 版本兼容性

本书描述了Swift 4.1,它是Xcode 9.2中包含的Swift的默认版本. 可以使用Xcode 9.2用Swift 4或Swift 3 build targets.

当Swift 4编译器使用Swift 3代码时,它将其语言版本识别为3.2. 因此,可以使用像#if swift(> = 3.2)
这样的条件编译块来编写与多个版本的Swift编译器兼容的代码.

当使用Xcode 9.2构建Swift 3代码时,大部分新的Swift 4功能都可用. 也就是说,以下功能仅适用于Swift 4代码:

  • 子字符串操作返回子Substring类型的实例,而不是String.
  • @objc属性隐式添加到更少的位置.
  • 对同一文件中的类型的扩展(Extensions)可以访问该类型的私有成员.

用Swift 4编写的target可以依赖于用Swift 3编写的target,反之亦然. 这意味着,如果有一个大型项目分为多个框架,可以一次将代码从Swift 3迁移到Swift 4.

Control Flow 控制流

使用if和switch来创建条件,并使用for-in,while和repeat-while来创建循环. 条件或循环变量的括号是可选的. 需要身体周围的大括号.

在if语句中,条件必须是Boolean表达式 - 这意味着如score {…}这样的代码是一个错误,而不是隐式的比较为零.

可以使用if和let使用可能会丢失的值. 这些值表示为可选项. 可选值包含一个值或包含nil以指示缺少一个值. 在值的类型后面写一个问号(?),将该值标记为可选.

练习: 把optionalName改成nil,greeting会是什么？添加一个else语句,当optionalName是nil时给greeting赋一个不同的值.

如果可选值为nil,则条件为false,并且花括号中的代码被跳过. 否则,在let之后,可选值将被解包并分配给该常量,这会使代码块中的解包值可用.

处理可选值的另一种方式是使用??提供默认值. 如果可选值缺失,则使用默认值.

Switches支持任何种类的数据和各种比较操作 - 它们不限于integers 和相等性测试.

请注意,let可以用于模式中,以将与模式匹配的值分配给常量.

在匹配的switch case内部执行代码后,程序从switch语句中退出. 执行不会延续到下一个案例,因此不需要在每个案例的代码末尾显式地跳出交换机.

通过提供一对用于每个键值对的名称,可以使用for-in来遍历字典中的项目. 字典是无序集合,所以它们的键和值以任意顺序迭代.

使用while来重复一段代码,直到条件改变. 循环的条件可以在最后,确保循环至少运行一次.

可以使用..<来创建循环中的索引,以创建一系列索引.

使用..<制作一个省略其上限值的范围,并使用…来制作一个包含两个值的范围.

Functions and Closures 函数和闭包

使用func来声明一个函数. 通过使用括号中的参数列表跟随其名称来调用函数. 使用->将参数名称和类型从函数的返回类型中分离出来.

默认情况下,函数使用它们的参数名称作为其参数的标签. 在参数名称之前写入自定义参数标签,或者写入_以使用不带参数标签.

使用一个tuple来创建一个复合值 - 例如,从一个函数返回多个值. tuple的元素可以通过名字或数字来引用.

函数可以嵌套. 嵌套函数可以访问在外部函数中声明的变量. 可以使用嵌套函数来重构一个太长或太复杂的函数.

函数是first-class type. 这意味着一个函数可以返回另一个函数作为它的值.

函数可以将另一个函数作为其参数之一.

函数实际上是一种特殊的闭包,可以使用{}来创建一个匿名闭包.使用in将参数和返回值类型声明与闭包函数体进行分离.

练习: 重写闭包,对所有奇数返回0.

有很多种创建闭包的方法.如果一个闭包的类型已知,比如作为一个回调函数,可以忽略参数的类型和返回值.单个语句闭包会把它语句的值当做结果返回.

可以通过参数位置而不是参数名字来引用参数——这个方法在非常短的闭包中非常有用.当一个闭包作为最后一个参数传给一个函数的时候,它可以直接跟在括号后面.

Objects and Classes 对象和类

使用class,然后使用类的名称创建一个类. 类中的属性声明与常量或变量声明的写法相同,唯一的区别就是它们的上下文是类. 同样,方法和函数声明的写法也是一样的.

练习: 使用let添加一个常量属性,再添加一个接收一个参数的方法.

通过在类名后加括号来创建一个类的实例. 使用点语法来访问实例的属性和方法.

Shape类的这个版本缺少一些重要的东西:一个构造函数来初始化类实例.使用init来创建一个构造器.

注意self被用来区别实例变量.当创建实例的时候,像传入函数参数一样给类传入构造器的参数.每个属性都需要赋值——无论是通过声明(就像numberOfSides)还是通过构造器(就像name).

如果需要在释放对象之前执行一些清理,使用deinit创建一个deinitializer析构函数.

子类的定义方法是在它们的类名后面加上父类的名字,用冒号分割.创建类的时候并不需要一个标准的根类,所以可以忽略父类.

子类如果要重写父类的方法的话,需要用override标记——如果没有添加override就重写父类方法的话编译器会报错.编译器同样会检测override标记的方法是否确实在父类中.

用于检测NFC(近场通讯) 标签和读取包含NDEF数据的信息.

Overview

你的app可以通过NFC读取标签,从而给予用户提供更多有关其物理环境和真实世界的对象内容信息.比如:你的app可能可以给用户提供商店里的商品信息或者是他们在博物馆参观时的陈列品的信息.

要求

  • SDK要求iOS 11.0+
  • 目前只有iPhone 7 and iPhone 7 Plus支持读取NFC NDEF标签
  • Supports ISO14443 and ISO15693 tags.

使用Core NFC,你可以读取到包含NDEF (NFC数据交换格式)格式数据的1到5类型的NFC 标签.你的app可以通过创建一个NFC NDEF reader session对象同时提供一个delegate代理来读取标签.当一个正在运行的reader session对象发现了一个包含NDEF信息的标签后,就会将这个信息传递给它的delegate代理.这个delegate代理除了能读取消息还能处理session变为无效的情况.
要启用您的应用程序来检测NFC标签,请打开Xcode项目中的Near Field Communication Tag阅读功能. 要了解有关向项目添加功能的更多信息,see Add a capability to a target.

Core NFC包含以下内容

  • Reader Sessions

    • class NFCNDEFReaderSession
      用于检测NFC NDEF标签的reader session类
    • protocol NFCReaderSessionProtocol
      定义了与reader session交互的通用接口
    • class NFCReaderSession
      用于描述一个用来检测NFC标签的reader session的抽象类

  • NFC Tags

    • protocol NFCTag
      定义了NFC或RFID(Radio Frequency Identification,射频识别)标签的属性
    • class NFCTagCommandConfiguration
      此类提供了一组的参数可以让你用来配置NFC标签命令

  • NDEF Messages

    • class NFCNDEFMessage
      NFC NDEF消息类,由一组payload记录组成
    • class NFCNDEFPayload
      此类表示在NFC NDEF消息里的一个payload记录
    • enum NFCTypeNameFormat
      NFC类型枚举,用于指定NFC NDEF消息中的内容类型.
      ​ NFCTypeNameFormatAbsoluteURI
      ​ The data uses the uniform resource identifier standard.
      ​ NFCTypeNameFormatEmpty
      ​ 有记录是空的,不包含任何信息
      ​ NFCTypeNameFormatMedia
      ​ The data uses the media type defined by RFC 2046.
      ​ NFCTypeNameFormatNFCExternal
      ​ The data contains a value that uses the NFC record type definition for external types.
      ​ NFCTypeNameFormatNFCWellKnown
      ​ The data contains a value that uses a type that follows the NFC record type definition.
      ​ NFCTypeNameFormatUnchanged
      ​ The data is not the first record in a series of records containing chunked data.
      ​ NFCTypeNameFormatUnknown
      ​ The data type is unknown.

  • Errors

    • enum NFCReaderError

了解 NDEF-Records

为了使用NFC-tags,理解NDEF(NFC Data Exchange Format)规范是非常重要的.每当CoreNFC发现一个新标签,didDetectNDEFs委托方法将提供一个NDEF messages（[NFCNDEFMessage]）.
通常,只包含一个NDEF message,但是规范保持灵活性,以便同时提供多个消息.
每个NFCNDEFMessage都包含一个payload-records([NFCNDEFPayload]),用于存放开发人员感兴趣的实际信息.目前,CoreNFC框架中有四个属性用于访问这些属性:

  1. typeNameFormat指定payload 是哪种NDEF 类型. 详见NFCTypeNameFormat NFC类型枚举,用于指定NFC NDEF记录中的数据类型.
  2. typepayload的类型.
  3. identifierpayload的唯一标识符.
  4. payloadpayload的数据,数据格式取决于 typeNameFormat 指定的类型.

用户体验

  1. 扫描 NDEF-tag 通常会在重新启动iPhone后立即执行一次.从那时起,它可能起作用或不起作用,通常不起作用,需要重新启动iPhone.在iOS 11下是出现过的.
  2. 如果扫描 RFID-tag 是最新的或者不包含 NDEF-tag(e.g. a credit-card),reader将会times out(error 201).
  3. 如果扫描 RFID-tag 包含加密分区,reader session将会throws error 200(readerSessionInvalidationErrorUserCanceled)

RFID功能 ⚠️ 警告 目前iOS11还不支持

我们使用NFCNDEFReaderSession来处理 NDEF NFC-Chips.在CoreNFC中实际还有另一个类,叫做NFCISO15693ReaderSession用来处理RFID-tags,
其规范为ISO15693,可以通过公布的delegates 和 NFCISO15693Tag class来进行处理.
你甚至可以发送自定义命令到 RFID-chip上,通过readerSession:didDetectTags: delegate 和NFCISO15693CustomCommandConfiguration class 来实现.
不过目前 iOS 11 Beta 1-2会抛出一个功能不支持的错误,Beta 3-4将打开扫描对话框,但立即出现一个abort的错误.
这可能会导致com.apple.developer.nfc.readersession.formats授权密钥内的另一个值.
使用ISO15693或RFID之类的东西到目前为止都不能工作,并且不能完成构建.我们看看未来Apple 是否在iOS 11 + Beta 更新此特性 🙂

Overview 概述

UIKit框架提供了为iOS和tvOS构建应用程序所需的核心对象. 您可以使用这些对象在屏幕上显示您的内容,与该内容进行交互,并管理与系统的交互. 应用程序依赖于UIKit的基本行为,UIKit提供了许多方法来定制该行为以满足您的特定需求.

Xcode提供模板项目作为您创建的每个应用程序的起点. 例如,图1显示了使用Xcode中的单一视图应用程序模板创建的应用程序的结构. 模板项目提供最小的用户界面,因此您可以立即构建和运行项目,并在设备或模拟器中查看结果.

The structure of a single view iOS app

当您构建应用程序时,Xcode会编译您的源文件并为您的项目创建一个应用程序包. 应用程序包是一个结构化的目录,包含与应用程序关联的代码和资源. 资源包括支持代码的image assets,storyboard文件,strings文件和app metadata. 应用程序包的结构很重要,但Xcode知道您的资源需要去哪里,所以不要担心.

  • App icons
  • Launch screen storyboard 启动屏 storyboard

您可以对Info.plist文件进行的一项常见修改是声明您的应用程序的硬件和软件要求. 这些要求是您如何向系统传达您的应用程序需要运行的方式. 例如,导航应用程序可能需要GPS硬件的存在才能提供转由转向的指示,如图3所示.App Store阻止将应用程序安装在不符合应用程序要求的设备上.

UIKit 框架为您的iOS或tvOS应用程序提供了所需的基础架构. 它提供了用于实现界面的窗口和视图体系结构,用于向应用程序提供多点触控和其他类型输入的事件处理基础结构,以及管理用户,系统和应用程序之间的交互所需的主运行循环. 该框架提供的其他功能包括动画支持,文档支持,绘图和打印支持,关于当前设备的信息,文本管理和显示,搜索支持,可访问性支持,应用程序扩展支持和资源管理.

  • First Steps
    • About App Development with UIKit
      了解UIKit和Xcode为您的iOS和tvOS应用程序提供的基本支持.
  • App Structure
    • Core App
      管理您的应用的数据模型及其与系统的交互.
    • Resource Management
      管理您存储在主要可执行文件之外的 images, strings, storyboards, and nib files
    • App Extensions
      将应用程序的基本功能扩展到系统的其他部分.
  • User Interface 用户界面
    • Views and Controls
      在屏幕上显示您的内容,并定义与该内容相关的交互.
    • View Controllers
      使用视图控制器管理你的界面,并促进周围的应用程序的内容导航.
    • View Layout
      使用堆栈视图自动布置界面的视图. 当您需要精确放置视图时,请使用自动布局.
    • Animation and Haptics 触觉
      使用基于视图的动画和触觉向用户提供反馈.
    • Windows and Screens
      为您的视图层次结构和其他内容提供一个容器.
  • User Interactions 用户交互
    • Touches, Presses, and Gestures 触摸,按压和手势
      在手势识别器中封装您的应用程序的事件处理逻辑,以便您可以在整个应用程序中重复使用该代码.
    • Drag and Drop 拖放
      通过使用交互API和视图,拖放到您的应用程序.
    • Focus Interactions 焦点互动
      使用远程或游戏控制器导航UIKit应用程序的界面.
    • Peek and Pop
      使用3D触摸输入来显示您的内容的自定义预览和操作.
    • Keyboard and Menus 键盘和菜单
      处理键盘输入,并显示自定义操作菜单.
    • Accessibility 无障碍
      让残障用户更容易使用您的应用.
  • Graphics, Drawing, and Printing
    • Images and PDF
      创建和管理图像,包括使用位图和PDF格式的图像.
    • Drawing
      使用渲染器配置应用程序的绘图环境,并绘制路径,字符串和阴影.
    • Printing
      显示系统打印面板并管理打印过程.
  • Text
    • Text Display and Fonts
      使用UIKit视图来显示文本,管理字体和检查拼写.
    • Text Storage
      管理文本存储,并协调文本的布局.
    • Keyboards and Input
      配置系统键盘,或创建自己的键盘和自己处理输入.
